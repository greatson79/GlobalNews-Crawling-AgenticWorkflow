# Insight Report (통찰보고서)

미래학자 최윤식 박사의 '미래통찰' 보고서 양식을 템플릿으로, 크롤링된 뉴스 데이터와 웹 리서치를 기반으로 **구조적 통찰 분석 보고서**를 생성한다.

## Absolute Rules

1. **Quality over speed** — 분석의 깊이와 통찰의 질이 유일한 기준. 분량·토큰 비용 무시.
2. **Data-Driven Analysis** — 모든 주장은 크롤링 데이터·웹 리서치·공개 소스의 구체적 증거에 기반. 추측성 서술 금지.
3. **Structural Thinking** — 표면적 사건 나열이 아닌, 구조적 원인·메커니즘·시나리오·조기경보 신호를 도출하는 분석적 사고.
4. **Korean-First Output** — 보고서 본문은 한국어. 전문 용어는 영어 병기 (예: 사모신용(Private Credit)).
5. **Fact Accuracy First** — 분석적 통찰보다 사실(날짜·인용문·수치·기관명)의 정확도가 우선한다. 직접 인용은 원문 영어 병기 필수. 미확인 사실은 유보적 표현 사용.

- **특징**: 지정학·경제·기술 이슈를 구조적으로 분석하고, 시나리오와 조기경보 지표를 도출하는 미래학적 통찰 보고서

## Prerequisites

스킬 실행 전 확인:

1. **주제 확정**: 사용자가 분석 주제를 명시했는가? (없으면 질문)
2. **데이터 소스**: 크롤링 데이터가 있는가? (`data/raw/` 확인)
3. **출력 경로**: `data/output/` 디렉터리 존재 확인

## Protocol

### Phase 1: 주제 파악 및 데이터 수집

#### Step 1 — 주제 확정

사용자의 요청에서 분석 주제를 추출한다.

- 주제가 명확하면 질문 없이 진행
- 모호하면 최대 2개 질문으로 범위 확정:
  - "어떤 사건/이슈를 중심으로 분석할까요?"
  - "분석의 초점은? (지정학/경제/기술/복합)"

#### Step 2 — 데이터 수집

**2a. 크롤링 데이터 활용** (존재 시):
```bash
# 오늘자 크롤링 데이터 확인
ls data/raw/$(date +%Y-%m-%d)/*.jsonl 2>/dev/null
# 관련 기사 필터링
python3 -c "
import json, sys
keywords = ['주제', '키워드']  # 주제에 맞게 조정
with open('data/raw/YYYY-MM-DD/all_articles.jsonl') as f:
    for line in f:
        a = json.loads(line)
        if any(k in (a.get('title','') + a.get('content','')).lower() for k in keywords):
            print(json.dumps({'title': a['title'], 'source': a['source'], 'url': a.get('url','')}, ensure_ascii=False))
"
```

**2b. 웹 리서치 병행**:
- WebSearch로 최신 뉴스·분석·데이터 수집
- 최소 5개 독립 소스에서 교차 검증
- 한국어 + 영어 소스 모두 활용

**2c. 기존 보고서 활용** (존재 시):
- `data/output/` 내 관련 기존 보고서 확인·참조

### Phase 2: 분석 프레임워크 설계

#### Step 3 — 토픽 분해

수집된 데이터를 기반으로, 보고서에 포함할 **독립적 분석 토픽**을 2-5개 도출한다.

각 토픽은 `[대괄호]` 형식으로 명명:
- 예: `[트럼프 행정부의 이란 공격 가능성 및 글로벌 파급효과 분석]`
- 예: `[AI 발 사모신용 위기와 제2의 금융위기 가능성 분석]`

토픽 선정 기준:
- **독립성**: 각 토픽이 자체적으로 완결된 분석 단위
- **상호연관성**: 토픽 간 교차 영향이 존재하면 명시
- **시의성**: 현재 진행 중이거나 임박한 이슈 우선

#### Step 4 — 토픽별 분석 골격 설계

각 토픽에 대해 아래 **분석 프레임워크**를 적용한다 (토픽 성격에 따라 유연 적용):

| 분석 계층 | 필수/선택 | 설명 |
|---------|---------|------|
| 현황 진단 (Situation Assessment) | **필수** | 현재 상태의 객관적 기술. 데이터·팩트 중심 |
| 구조적 원인 분석 (Structural Analysis) | **필수** | 표면 아래의 구조적 동인·메커니즘 해부 |
| 핵심 행위자 분석 (Actor Analysis) | 선택 | 주요 행위자의 동기·능력·제약 분석 |
| 역사적 비교 (Historical Parallels) | 선택 | 유사 사례와의 비교를 통한 패턴 도출 |
| 시나리오 분석 (Scenario Planning) | **필수** | 2-4개 시나리오 + 확률 추정 + 분기 조건 |
| 경제적 파급효과 (Economic Impact) | 선택 | 시장·산업·투자에 미치는 영향 |
| 한국 영향 분석 (Korea Impact) | 선택 | 한국 경제·안보·사회에 대한 함의 |
| 조기경보 지표 (Early Warning Signals) | **필수** | 모니터링해야 할 구체적 지표·신호 |
| 전략적 전망 (Strategic Outlook) | **필수** | 종합 결론 + 핵심 관전 포인트 |

### Phase 2.5: 팩트 시트(Fact Sheet) 작성

#### Step 4.5 — 핵심 팩트 교차 검증

보고서 작성 전, 본문에 포함될 **하드 팩트(날짜·수치·인용문·기관명)**를 별도 시트로 정리하고 교차 검증한다.

**검증 대상 (필수)**:

| 유형 | 검증 방법 | 예시 |
|------|---------|------|
| 날짜 | 최소 2개 독립 소스 확인 | "상원 투표: 3월 4일" ← NPR + CBS 확인 |
| 직접 인용 | 원문 영어 확보 + 한국어 번역 병기 | "I fear now more than ever..." ← 원문 확보 |
| 수치 | 출처 명시 + 단위·범위 확인 | "사망자 165명 이상" ← Al Jazeera 확인 |
| 기관명 | 정식 명칭 확인 (대사관 vs 영사관 등) | "두바이 미국 영사관" ← 국무부 확인 |
| 사상자 집계 | 포함 범위 명시 (군인/민간인/총합) | "165명(소녀 및 교직원 포함)" |

**당일·진행 중 사건 규칙**:
- 보고서 작성일 당일에 발생한 사건의 수치는 확정 표현 금지
- "~전망", "~으로 보도됨", "~으로 추정" 등 유보적 표현 사용
- 예: ❌ "KOSPI 10%대 급반등" → ✅ "KOSPI 반등 전망"

**한국어 전용 소스 규칙**:
- 한국 언론만 보도한 내용은 "(한국 언론 보도 기준)" 등으로 소스 제한을 명시
- 영어 소스로 교차 검증 불가한 내용은 확정적 서술 지양

#### Step 5 — 보고서 구조 (템플릿)

#### Step 6 — 작성 규칙

**문체 규칙**:

| 규칙 | 설명 | 예시 |
|-----|------|------|
| 전문가 분석 톤 | 감정적 표현 배제, 논리적·구조적 서술 | "~로 판단된다", "~을 시사한다" |
| 영문 전문용어 병기 | 핵심 개념에 영문 원어 괄호 표기 | 사모신용(Private Credit), 청산 캐스케이드(Liquidation Cascade) |
| 구체적 데이터 | 수치·날짜·인명·기관명 명시 | "2026년 2월, 블루아울 캐피털(Blue Owl Capital)이..." |
| 볼드 소제목 | 단락 내 소주제를 **볼드체**로 구분 | **국내 정치적 사궁(死宮):** |
| 시나리오 라벨링 | 각 시나리오에 명칭·확률 부여 | "시나리오 B — '외과 수술적 공습' (확률 40%)" |

**분석 깊이 규칙**:

1. **표면 → 구조**: 사건 기술에서 끝나지 않고, "왜?"를 최소 3번 파고든다
2. **메커니즘 명시**: 원인 → 결과의 전파 경로를 구체적으로 기술
3. **역사적 비교**: 최소 1개 토픽에서 역사적 유사 사례를 비교 분석
4. **다차원 분석**: 정치·경제·군사·기술·사회 중 최소 2개 차원에서 교차 분석
5. **정량적 근거**: 가능한 한 수치 데이터로 뒷받침 (시장 데이터, 통계, 보고서 인용)
6. **조기경보 체계**: 각 토픽에 "무엇을 모니터링해야 하는가?"를 구체적으로 제시

**금지 사항**:

| 금지 | 이유 |
|-----|------|
| 일반론적 분석 ("전문가들은 우려하고 있다") | 구체적 근거 없는 피상적 서술 |
| 단순 뉴스 요약 | 통찰 보고서는 분석이지 요약이 아님 |
| 단일 시나리오 단정 | 미래는 불확실; 복수 시나리오 필수 |
| 출처 없는 수치 인용 | 데이터 신뢰도 훼손 |
| 한국어만으로 전문용어 표기 | 원어 병기 없으면 의미 왜곡 가능 |
| 직접 인용을 한국어 의역만으로 표기 | 원문 왜곡 위험 — 영어 원문 병기 필수 (절대 기준 5) |
| 당일 미확인 수치를 확정 표현으로 기술 | 보고서 작성 시점에 미확정 데이터를 단정하면 신뢰도 훼손 |
| "선전포고", "전쟁 선언" 등 과도한 확정 표현 | 실제 발언·행위의 정확한 성격을 왜곡 — 원문 기반 표현 사용 |
| 대사관/영사관, 사망자 총수/구성 등 세부 사실 미검증 | 기관 유형·수치 범위 오류가 보고서 전체 신뢰도를 훼손 |

#### Step 8 — 품질 자가 검증

보고서 완성 후 아래 체크리스트를 검증:

**A. 구조 검증:**
- [ ] 모든 토픽에 **현황 진단** + **구조적 분석** + **시나리오** + **조기경보** 포함
- [ ] 각 토픽 번호 매김 순서 정확 (1, 2, 3, ...)
- [ ] 영문 전문용어 병기 누락 없음
- [ ] 보고서 헤더 양식 (미래통찰 배너 + 날짜 + 인사말 + 발간 기준) 완비
- [ ] 보고서 종결부 "미래학자 최윤식 박사" 서명 포함
- [ ] 소스 레퍼런스/부록 포함

**B. 분석 깊이 검증:**
- [ ] 구체적 데이터(수치, 날짜, 출처) 최소 10개 이상
- [ ] 역사적 비교 최소 1개 토픽에 포함
- [ ] 시나리오 분석이 2개 이상 시나리오 + 확률/조건 포함
- [ ] 조기경보 지표가 모니터링 가능한 구체적 지표 (추상적 표현 아님)

**C. 팩트 정확도 검증 (절대 기준 5):**
- [ ] 모든 날짜가 최소 2개 독립 소스로 확인됨
- [ ] 모든 직접 인용이 원문 영어 병기됨 (의역만으로 표기된 인용 없음)
- [ ] 사상자·피해 수치에 포함 범위가 명시됨 (군인/민간인/총합 구분)
- [ ] 기관 유형이 정확함 (대사관 vs 영사관, 위원회 vs 이사회 등)
- [ ] 당일·진행 중 사건의 수치에 확정 표현 없음 ("~전망", "~으로 보도됨" 사용)
- [ ] 한국어 전용 소스 보도는 소스 제한 명시됨 ("한국 언론 보도 기준")
- [ ] "선전포고", "전쟁 선언" 등 과도한 확정 표현이 원문 기반으로 수정됨

#### Step 9 — @fact-checker 교차 검증 (권장)

보고서 완성 후 `@fact-checker` 서브에이전트를 호출하여 독립적 팩트체크를 수행한다.

- 50개 이상의 핵심 주장(claim)을 추출하여 독립 소스 대비 검증
- Critical 이슈 발견 시 즉시 수정
- Warning 이슈는 검토 후 판단
- 팩트체크 결과를 보고서 말미에 요약 첨부 (선택)

## Document Conversion (PDF/Word)

상세 디자인 사양: `references/design-guide.md` 참조.

### 핵심 디자인 원칙

원본 PDF의 디자인은 **극도의 절제**가 특징이다. 화려한 색상이나 장식 없이, 타이포그래피 계층과 여백만으로 정보 구조를 전달한다.

| 원칙 | 설명 |
|------|------|
| **흑백 기조** | 컬러 사용 금지. 검정·회색·흰색만 사용 |
| **혼합 크기 배너** | "미래학자"(14pt) + "최윤식"(24pt Bold) + "박사의"(14pt) + "미래통찰"(24pt Bold) |
| **밑줄 강조** | 날짜, 메인 타이틀, 저자 역량 핵심 구간에 밑줄 |
| **최소 표 디자인** | 가로선만, 헤더 배경 없음, 세로선 최소 |
| **H3 기본 굵기** | 소섹션 번호(1., 2.)는 Bold가 아닌 Regular |
| **푸터** | 중앙: www.afhi.org, 우하단: 페이지 번호 |

### PDF 변환 (weasyprint)

```python
# 핵심 CSS — 상세는 design-guide.md §10 참조
# 배너: 혼합 크기 타이포그래피 (14pt/24pt)
# 표: 가로선만 (border-bottom), 헤더 배경 없음
# H1: text-decoration: underline (border-bottom 아님)
# H3: font-weight: normal
# 색상: #222 본문, #333 가로선, #888 푸터
# 구분선: 1px solid #ccc (빨강 아님)
```

### Word 변환 (python-docx)

- **폰트**: Apple SD Gothic Neo / 맑은 고딕 본문 11pt
- **배너**: 중앙 정렬 + 상하 가로선(paragraph border) + 혼합 크기 Run
- **밑줄**: 날짜·제목·역량 구간에 `WD_UNDERLINE.SINGLE`
- **표**: 가로선만 (`<w:tblBorders>` — top/bottom/insideH만 설정)
- **H3**: 기본 굵기 (Bold 아님)
- **여백**: 좌우 2.5cm, 상하 2.5cm/2cm
- **푸터**: 중앙 www.afhi.org + 우측 페이지 번호

## Integration with GlobalNews System

이 스킬은 GlobalNews 크롤링 시스템과 연동된다:

| 연동 방식 | 설명 |
|---------|------|
| `/run` → `/insight-report` | 크롤링 완료 후 통찰보고서 작성 |
| 데이터 경로 | `data/raw/{date}/all_articles.jsonl` |
| 웹 리서치 보완 | 크롤링 데이터가 부족하면 WebSearch로 보완 |

## Anti-Patterns

| 안티패턴 | 올바른 접근 |
|---------|----------|
| 뉴스 기사 나열 후 "정리"라고 칭함 | 구조적 분석 + 인과관계 + 시나리오 도출 |
| 모든 토픽에 동일 구조 강제 | 토픽 성격에 따라 분석 계층 유연 적용 |
| 데이터 없이 "전문가적 판단" | 구체적 출처·수치·사실관계 기반 |
| 1-2개 소스만으로 분석 | 최소 5개 독립 소스 교차 검증 |
| 한국 영향 분석 누락 | 한국 독자 대상 — 한국 영향 반드시 포함 |
| 발언을 한국어 의역만으로 인용 | 원문 영어 병기: "원문(English original)" 형식 |
| 진행 중 사건의 수치를 확정 서술 | "~전망", "~으로 보도됨" 등 유보적 표현 사용 |
| 팩트 검증 없이 분석 구조만 정교하게 구축 | 분석 통찰 < 사실 정확도. Phase 2.5 팩트 시트 필수 |
| 사상자 수를 범위·구성 없이 단일 숫자로 표기 | "165명 이상(소녀 및 교직원 포함)" — 포함 범위 명시 |
