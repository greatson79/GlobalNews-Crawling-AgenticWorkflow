# 분석 프레임워크 (Analysis Framework)

> 최윤식 박사의 통찰보고서에서 추출한 분석 기법과 사고 패턴.

## 1. 핵심 분석 기법 (Core Analytical Techniques)

### 1.1 구조적 해부 (Structural Dissection)

표면적 사건을 넘어 **구조적 메커니즘**을 노출하는 기법.

**패턴**: "왜?" → "어떤 구조가?" → "누가 이득?" → "어디로 향하나?"

**실례 (PDF p.1-2)**:
- 사건: 미-이란 군사적 긴장 고조
- 구조: 미 해군 전력 1/2 + 해병 원정 상륙군 + F-22/F-35/B-2 전진 배치
- 메커니즘: "공세적 억지" → "즉각 대응 태세(Ready-to-Strike)" 전환
- 동인: 정치적 사궁(지지율 급락) + 11월 중간선거 + 협상 전술

### 1.2 다차원 교차 분석 (Cross-Dimensional Analysis)

하나의 사건을 **정치·경제·군사·기술·사회** 다중 렌즈로 분석.

**실례 (이란 공격 분석)**:
- 군사: 4단계 타격 전략 (SEAD → 지휘통제 → 호르무즈 → 핵 시설)
- 정치: 11월 중간선거 + '애드벌룬' 전술 + 이스라엘 예산안
- 경제: 유가 급등 → 인플레이션 → M2 확장 → GDP 명목 성장
- 외교: 제네바 핵 협상 + "외교 외의 선택지" 경고

### 1.3 역사적 비교 분석 (Historical Parallels)

현재 사건과 역사적 선례를 병치하여 **패턴과 차이**를 도출.

**실례 (PDF p.7)**:
- 태종 이방원 vs 시진핑: 공신 숙청 비교
  - 공통점: 권력 기반 제거
  - 차이점: 태종은 후계 안정 위해, 시진핑은 영구 집권 위해
  - 통찰: "태종은 피를 흘려 세종을 얻었으나, 시진핑은 피를 흘려 거대한 진공(Vacuum)을 만들고 있다"
- 마오쩌둥 우회 전략 재현: 관료 체계 우회 → 일선 병사 직접 호소

### 1.4 시나리오 분기 분석 (Scenario Branching)

**2-4개 시나리오**를 정의하고 각각의 **확률·조건·결과**를 추정.

**구성 요소**:
- 시나리오 명칭 (직관적 라벨)
- 발생 조건 (무엇이 트리거?)
- 확률 추정 (근거 포함)
- 결과 체인 (1차 → 2차 → 3차 효과)
- 분기 시점 (언제 어느 시나리오로 분기?)

### 1.5 조기경보 지표 도출 (Early Warning Signal Extraction)

각 분석의 마지막에 **모니터링 가능한 구체적 지표**를 제시.

**좋은 조기경보 지표의 조건**:
- ✅ 관찰 가능 (공개 데이터, 보도, 시장 데이터)
- ✅ 구체적 (날짜, 수치, 행위자 명시)
- ✅ 선행적 (사건 발생 전 변화 감지 가능)
- ❌ "전문가들의 우려가 커지면" (추상적, 측정 불가)

**실례**:
- "향후 3월 31일(네타나후 예산안 마감)과 3월 중순(노루즈 설날) 사이 무력 충돌 여부"
- "현금흐름표 상의 CapEx: AI 시설 투자에 따른 현금 고갈 여부"
- "PIK 전환율 및 조달 금리 스프레드"
- "IBIT 옵션 미결제 약정 블록 분포 (6만~7만 5천 달러 구간)"

### 1.6 전파 메커니즘 모델 (Propagation Mechanism)

위기의 **전파 경로**를 단계별로 추적하는 기법.

**실례 (PDF p.11 — 9단계 전파 모델)**:
```
State 0: 개별 펀드 유동성 경색 (Blue Owl)
  → State 1: SaaS 산업 매출 급감 (AI 파괴)
  → State 2: 대출 원리금 상환 실패
  → State 3: 자산 가치 급락 (미실현 자산)
  → State 4: 네트워크 전이 (Ares, Apollo, Blackstone)
  → ...
  → State 8: 정상 상태 복귀
```

### 1.7 행위자 동기 분석 (Actor Motivation Analysis)

핵심 행위자의 **동기·능력·제약**을 삼각 분석.

| 분석 축 | 질문 |
|--------|------|
| 동기 (Motive) | 이 행위자가 원하는 것은? 정치적·경제적·이념적 목표는? |
| 능력 (Capability) | 실행할 자원·기술·동맹이 있는가? |
| 제약 (Constraint) | 무엇이 행동을 억제하는가? 시간·비용·내부 반대? |

## 2. 분석 품질 기준 (Quality Criteria)

### 통찰의 5가지 수준

| 수준 | 설명 | 보고서에서의 위치 |
|------|------|-------------|
| Level 1: 사실 기술 | "무슨 일이 일어났나" | 현황 진단 (필수이나 여기서 멈추면 안 됨) |
| Level 2: 원인 분석 | "왜 일어났나" | 구조적 원인 분석 |
| Level 3: 패턴 인식 | "이전에도 이런 일이 있었나" | 역사적 비교 |
| Level 4: 시나리오 | "앞으로 어떻게 될 수 있나" | 시나리오 분석 |
| Level 5: 조기경보 | "무엇을 주시해야 하나" | 조기경보 지표 |

**최소 기준**: 모든 토픽이 Level 3 이상 도달. Level 1-2에서 멈추는 토픽은 통찰보고서 품질 미달.

### 데이터-주장 연결 규칙

모든 핵심 주장은 아래 구조를 따른다:

```
[데이터/사실] → [해석/메커니즘] → [함의/전망]
```

- **데이터**: 수치, 날짜, 인명, 기관명, 보도 출처
- **해석**: "이것이 의미하는 바는..."
- **함의**: "따라서 향후..."

## 3. 토픽 유형별 분석 가이드

### 지정학·안보 이슈
- 필수: 군사 역량 비교, 행위자 동기, 시나리오 분기, 핵심 날짜/데드라인
- 깊이: 전략적 의도 → 전술적 옵션 → 역사적 선례 → 시나리오

### 경제·금융 이슈
- 필수: 시장 데이터, 전파 메커니즘, 구조적 취약성, 모니터링 지표
- 깊이: 트리거 사건 → 전파 경로 → 시스템 리스크 → 투자 함의

### 기술·산업 이슈
- 필수: 기술 파괴 메커니즘, 산업 구조 변화, 승자/패자 분석
- 깊이: 기술 변화 → 비즈니스 모델 파괴 → 산업 재편 → 고용/사회 영향

### 정치·거버넌스 이슈
- 필수: 권력 구조 분석, 제도적 메커니즘, 역사적 비교
- 깊이: 표면 사건 → 권력 역학 → 제도적 취약성 → 장기 시나리오
